LLD Restaurant Supply API

Complete REST API reference for the LLD Restaurant Supply platform. Supports web ordering, voice AI integration, inventory management, invoicing, and staff operations.

Base URL
https://rs-lld-website-production.up.railway.app/api
Authentication
The API supports two authentication methods depending on the client type.

🔐 Session-based Auth (Web / Staff Portal)

Use POST /api/login to authenticate. A session cookie is set automatically. Include credentials in all subsequent requests (credentials: 'include' in fetch).

🔑 API Key Auth (Voice AI / External Integrations)

Pass your API key in the X-API-Key header or as the api_key query parameter. Keys are prefixed sk_live_ (production) or sk_test_ (sandbox). Generate keys in the Staff Portal → API Keys tab.

👤 User & Authentication
Register, login, and manage user accounts.
POST /api/register Register a new customer account ▶

Creates a new customer account. Returns the user object and sets a session cookie.

Request Body (JSON)
FieldTypeDescription
username*stringUnique display name
email*stringEmail address
password*stringMin 8 characters
company_nameoptstringRestaurant / business name
phoneoptstringPhone number (used for voice AI lookup)
Example Response 201
{ "id": 42, "username": "mario_kitchen", "email": "mario@example.com", "company_name": "Mario's Italian Kitchen" }
POST /api/login Authenticate and start a session ▶

Authenticates a user and sets a session cookie. Works for both customer and staff accounts.

Request Body (JSON)
FieldTypeDescription
email*stringRegistered email address
password*stringAccount password
Example Response 200
{ "message": "Login successful", "user": { "id": 42, "username": "mario_kitchen", "role": "customer" } }
GET /api/me Get current authenticated user 🔐 Auth required ▶

Returns the profile of the currently authenticated user based on session cookie.

Example Response 200
{ "id": 42, "username": "mario_kitchen", "email": "mario@example.com", "company_name": "Mario's Italian Kitchen", "phone": "702-555-0100" }
📦 Orders
Create and manage customer orders. Supports web, voice, and staff-entered orders.
GET /api/orders List orders 🔐 Auth required ▶

Returns orders for the current user. Staff/admin accounts see all orders with optional filters.

Query Parameters
ParamTypeDescription
statusstringFilter by status: pending, confirmed, packed, shipped, delivered, cancelled
pageintegerPage number (default: 1)
per_pageintegerResults per page (default: 20)
searchstringSearch by order number, customer name, or city
POST /api/orders Create a new order 🔐 Auth required ▶

Creates a new order for the authenticated customer. Triggers n8n order notification webhook automatically.

Request Body (JSON)
FieldTypeDescription
items*arrayArray of order items (see below)
delivery_address*stringStreet address for delivery
delivery_city*stringCity
delivery_stateoptstringState (default: NV)
delivery_zipoptstringZIP code
payment_methodoptstringnet_30, net_15, cod, credit_card (default: net_30)
special_notesoptstringDelivery or order notes
Item Object
FieldTypeDescription
product_id*integerProduct ID from /api/products
quantity*integerQuantity to order
Example Response 201
{ "order_id": 1234, "order_number": "LLD-2026-1234", "status": "pending", "total": 285.50 }
PUT /api/orders/<order_id>/status Update order status 🔐 Staff only ▶

Updates the status of an order. Staff and admin only.

Request Body (JSON)
FieldTypeDescription
status*stringpending | confirmed | packed | shipped | delivered | cancelled
staff_notesoptstringInternal notes visible to staff
🛒 Products
Browse and search the product catalog. No authentication required for read operations.
GET /api/products List all products ▶

Returns paginated product catalog with optional filtering by category or search term.

Query Parameters
ParamTypeDescription
category_idintegerFilter by category ID
categorystringFilter by category name (partial match)
searchstringFull-text search on name, SKU, brand
pageintegerPage number (default: 1)
per_pageintegerResults per page (default: 100, max: 500)
Example Response 200
{ "products": [{ "id": 1, "name": "Cambro 6-Pan Container", "sku": "CAM-6PAN", "unit_price": 42.50, "bulk_price": 38.00, "in_stock": true }], "total": 142, "page": 1 }
GET /api/products/<product_id> Get a single product ▶

Returns full details for a single product including images, pricing tiers, and stock status.

🎙️ Voice AI
Endpoints used by the ElevenLabs voice agent for phone-based ordering. Requires API key authentication.
POST /api/voice/get_customer Look up customer by phone number 🔑 API Key ▶

Called by the voice agent at the start of a call to identify the caller and retrieve their order history.

Request Body (JSON)
FieldTypeDescription
phone*stringCaller's phone number (E.164 or local format)
Example Response 200
{ "found": true, "customer_name": "Mario Rossi", "company": "Mario's Italian Kitchen", "last_order": "LLD-2026-1198", "credit_status": "good" }
POST /api/voice/get_products Search products for voice agent 🔑 API Key ▶

Returns matching products for a spoken product query. Optimized for voice — returns concise, speakable results.

Request Body (JSON)
FieldTypeDescription
query*stringNatural language product search (e.g. "cambro containers")
limitoptintegerMax results (default: 5)
POST /api/voice/place_order Place an order via voice 🔑 API Key ▶

Places an order on behalf of a customer identified by phone number. Supports both existing customers and guest orders. Triggers SMS payment link and n8n notification automatically.

Request Body (JSON)
FieldTypeDescription
phone*stringCustomer phone number
items*arrayArray of {product_id, quantity}
delivery_addressoptstringRequired for guest orders
payment_methodoptstringnet_30 (default), cod, credit_card
special_notesoptstringAny special delivery instructions
Example Response 201
{ "order_number": "LLD-2026-1235", "total": 185.00, "sms_sent": true, "payment_link": "https://buy.stripe.com/..." }
POST /api/voice/check_order_status Check order status by phone 🔑 API Key ▶

Returns the most recent order status for a customer identified by phone number.

Request Body (JSON)
FieldTypeDescription
phone*stringCustomer phone number
order_numberoptstringSpecific order number to check
POST /api/voice/cancel_order Cancel an order via voice 🔑 API Key + Admin ▶

Cancels an order. Only orders in pending or confirmed status can be cancelled. Admin authorization required.

Request Body (JSON)
FieldTypeDescription
order_number*stringOrder number to cancel
phone*stringCustomer phone for verification

📊 Inventory
Track and update product stock levels. Staff authentication required.

Returns current stock levels for all products, including low-stock alerts.

PUT /api/inventory/<product_id> Update stock level 🔐 Staff ▶

Updates the stock quantity and reorder threshold for a product.

Request Body (JSON)
FieldTypeDescription
quantity*integerCurrent stock quantity
reorder_pointoptintegerTrigger reorder alert below this level
in_stockoptbooleanOverride in-stock flag
💳 Payment Links
Generate Stripe payment links and send them via SMS to customers.
POST /api/payment-links/create Create and send a payment link 🔐 Staff ▶

Creates a Stripe payment link for an order and sends it to the customer via SMS (Twilio).

Request Body (JSON)
FieldTypeDescription
order_id*integerOrder to generate payment link for
phoneoptstringOverride phone number for SMS delivery
Example Response 200
{ "payment_url": "https://buy.stripe.com/test_abc123", "sms_sent": true, "phone": "+17025550100" }
🌐 Sourcing
Manage suppliers, request-for-quotes (RFQs), shipments, and QC inspections.
GET /api/sourcing/suppliers List suppliers 🔐 Staff ▶

Returns all registered suppliers with contact info and performance metrics.

POST /api/sourcing/rfqs Create a Request for Quote 🔐 Staff ▶

Sends an RFQ to one or more suppliers for specified products.

Request Body (JSON)
FieldTypeDescription
supplier_ids*arrayList of supplier IDs to send RFQ to
items*arrayProducts and quantities needed
deadlineoptstringISO date for quote deadline
POST /api/sourcing/orchestrate AI-powered sourcing orchestration 🔐 Staff ▶

Uses AI to analyze inventory needs, identify best suppliers, and recommend optimal sourcing actions.

📈 Voice Analytics
Monitor voice agent performance and manage call review queues.
GET /api/staff/voice/performance Get voice agent performance metrics 🔐 Staff ▶

Returns aggregated performance metrics for the voice AI agent: call volume, order conversion rate, average call duration, and revenue generated.

Query Parameters
ParamTypeDescription
daysintegerLookback period in days (default: 7)
GET /api/staff/voice/review-queue Get calls pending review 🔐 Staff ▶

Returns calls that have been flagged for human review — low confidence orders, complaints, or escalations.

POST /api/staff/voice/review-queue/<call_id>/approve Approve a flagged call 🔐 Staff ▶

Approves a flagged call, confirming the order and removing it from the review queue.

⚙️ Utilities
Health check and AI-powered utility endpoints.
GET /api/ping Health check ▶

Lightweight keep-alive endpoint. Returns 200 OK with a status message. Used to prevent Railway cold starts.

Response
{ "status": "ok" }
POST /api/save-me-money AI cost optimization analysis 🔐 Auth required ▶

Analyzes a customer's order history and identifies opportunities to save money through bulk pricing, product substitutions, or ordering pattern optimization.

Request Body (JSON)
FieldTypeDescription
customer_idoptintegerCustomer to analyze (defaults to current user)
monthsoptintegerLookback period in months (default: 3)
POST /api/chat AI chat assistant ▶

Conversational AI assistant for product questions, order help, and general inquiries. Powers the website chatbot.

Request Body (JSON)
FieldTypeDescription
message*stringUser's message
languageoptstringen or zh (default: en)
historyoptarrayPrevious messages for context
